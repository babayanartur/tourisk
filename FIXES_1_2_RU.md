# Tourisk Mobile 1.2.1

## Исправление запуска

- Удалён конфликт между `app.json` и `app.config.js`.
- Версии Expo SDK и нативных пакетов выровнены.
- Удалён дубликат `expo-font`, который попадал через несовместимую версию иконок.
- `npm start` очищает `.expo` и Metro-кэш, определяет LAN-IP компьютера и создаёт новый локальный QR.
- Backend URL автоматически становится `http://LAN-IP:8000`.

## Игровой контент

- Mobile берёт фигурки, достижения и места с backend.
- Любая новая фигурка из админки работает без обновления массива в приложении.
- Backend-изображение имеет приоритет. Локальные PNG оставлены только как аварийный fallback при недоступности сервера.
- Изменения контента кэшируются для офлайн-запуска.
- Открытия мест и карты синхронизируются с аккаунтом.

## Карта

- Карта изначально закрыта туманом.
- Туман построен из перекрывающихся облачных форм и медленно движется.
- Область раскрывается автоматически вокруг GPS-позиции.
- Фигурка анимируется только на карте; на главной, в профиле и рейтинге она статична.
- Масштаб и свечение фигурки приходят из админки.
- В Expo Go работает foreground GPS. Для фоновой геолокации нужен development build.

## Запуск

Сначала запустите backend на порту 8000, затем:

```bash
npm install
npm start
```

Сканируйте новый QR из терминала, а не старую запись Tourisk из истории Expo Go.
