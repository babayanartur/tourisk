# TOURISK PRINCIPLES

Version 1.0

---

## Mission

Rediscover curiosity through the real world.

Everything we build must strengthen this mission.

If a feature doesn't improve exploration,
it doesn't belong in Tourisk.

---

## t First

We don't build features.

We build experiences.

Every screen should make people want to explore.

---

## Simplicity Wins

Simple ideas executed perfectly
always beat complicated ideas executed poorly.

---

## Quality Over Speed

Fast is good.

Correct is better.

Polished is best.

---

## One Step Rule

One finished step.

One tested step.

One committed step.

Then move forward.

Never skip.

---

## MVP First

Finish the MVP before expanding.

Ideas are stored.

Execution comes first.

---

## Player First

Every decision must answer one question:

"Does this make exploration more exciting?"

If not —

don't build it.

---

## Consistency

Everything should feel like one world.

Same visual language.

Same feeling.

Same philosophy.

---

## Think Long-Term

We don't build for tomorrow.

We build foundations that can survive years.

---

## Our Standard

We don't compare ourselves with others.

We compare today's Tourisk
with yesterday's Tourisk.

Progress is the only competition.
