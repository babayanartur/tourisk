# Tourisk Backend 1.2.0

## Что изменено

- Фигурки, достижения и места являются backend-контентом и управляются из админки.
- Изображения загружаются в `uploads/pawns`, `uploads/achievements`, `uploads/places`.
- Поддерживаются PNG, JPEG и WEBP до 12 МБ.
- API возвращает абсолютный `imageUrl`; для production задайте `PUBLIC_URL`.
- При замене или удалении записи старый управляемый файл удаляется.
- При ошибке создания новый загруженный файл очищается, чтобы не копить мусор.
- Стандартные PNG-фигурки физически находятся на backend и автоматически копируются при первом запуске.
- Удалённый через админку контент не возвращается после перезапуска. Ручной `npm run seed` восстанавливает стандартный набор намеренно.
- Условия фигурок и достижений считаются на backend: территории, города, страны, XP, уровень, дистанция, площадь, места, звёзды и серия дней.
- Открытия мест, награды, XP, карта и выбранная фигурка синхронизируются с аккаунтом.
- Добавлены статистика, проверка здоровья и защищённая админка.

## Запуск

```bash
cp .env.example .env
npm install
npm start
```

Админка: `http://localhost:8000/admin`

По умолчанию для локальной разработки:

```text
login: admin
password: tourisk1111
```

На сервере обязательно замените `JWT_SECRET`, `ADMIN_JWT_SECRET` и `ADMIN_PASSWORD`.
